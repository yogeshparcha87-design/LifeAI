# LifeAI Security Policy

## Supported Versions

The following versions currently receive security updates.

| Version | Supported |
|---------|-----------|
| 1.0.x | ✅ Yes |
| 0.x.x | ⚠️ Development Only |

---

## Reporting a Security Vulnerability

If you discover a security vulnerability, please do **not** publish it publicly.

Instead:

1. Create a private security report (if available).
2. Contact the project maintainer.
3. Include detailed steps to reproduce the issue.
4. Allow time for the issue to be investigated and fixed before public disclosure.

---

## Security Goals

LifeAI is designed with security and privacy as core principles.

The project aims to:

- Protect user privacy.
- Encrypt sensitive data where applicable.
- Store only necessary information.
- Never expose confidential information without user permission.
- Follow secure coding practices.
- Regularly review dependencies for vulnerabilities.

---

## User Privacy

LifeAI should:

- Request permission before accessing device features.
- Minimize data collection.
- Allow users to delete their personal data.
- Be transparent about stored information.

---

## Secure Development Practices

Developers should:

- Never hardcode passwords or API keys.
- Use environment variables for secrets.
- Validate all user input.
- Keep dependencies updated.
- Follow secure authentication practices.
- Test security before releasing new versions.

---

## Third-Party Services

When integrating external services:

- Use official APIs.
- Follow each provider's security guidelines.
- Store API keys securely.
- Never expose secret keys in the application.

---

## Responsible Disclosure

Please report security issues responsibly.

The project maintainer will:

- Acknowledge the report.
- Investigate the issue.
- Release a fix when appropriate.
- Credit the reporter (if desired).

---

## Contact

For security-related concerns, please contact the project maintainer through GitHub.

---

Thank you for helping keep LifeAI secure. 🔒
